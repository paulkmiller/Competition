The Competition


PadMapper

Tech

  • Sources data primarily from Craigslist, packages it—and data from other services—into an attractive format
    Can sometimes list properties that don't actually exist; their sanitation & verification leaves much to be desired
  • Gets data from big management companies as well as individual landlords looking to list spare bedrooms
  • Built using nothing more than Bootstrap and jQuery + Google Maps

Design

  • The whole service is a top-down Google Map, and you can quickly jump between a location's Street View, WalkScore, and photos without the main page

Special

  • Can set up alerts for new listings and that the data is updated in real-time, which has been the key to their success

Hotpads

Tech

  • Pulls listing information from sites like Craigslist + other major listing services w/ commercial properties to represent
  • Street view for every listing
  • Built using React.js
  • Can save favorite listings

Design

  • Interface is uncomplicated, smooth, and fast
  • Provides helpful guides to help renters make informed choices
  • Properties in list-view highlight corresponding map icon & visa versa
  • Organization of information is visually appealing and easy to digest

Special

  • Specific maps for low-incoming housing or foreclosures
  • Allows users to contact properties and landlords right through the search map, while highlighting information like rental prices, photos ( including floorpans ), and more information once you click on a map marker ( property ). This is a very information dense website and packages it cleanly
  • Lists a properties' corresponding school zones, along with associated primary / middle / highschool based on property location

Lovely

Tech

  • Another Craigslist-sourced, top-down map-based apartment search tool
  • Built using Angular & Moment.js
  • Stores application information after the first submission so applying for multiple properties is easier
  • Offers rent auto-payment if the apartment / housing community supports it.

Design

  • Organizes listings into clusters instead of inundating the map with a ton of markers; a smart idea toward keeping # of assets on the DOM at a minimum
  • Two kinds of markers— red & blue. Red represents listings that have been posted in the past few hours
  • Geo-locator within search bar as clickable icon to automatically filter properties within present location
  • "Help & Feedback" redirects to a "Contact Support" window along with FAQ section to read while waiting
    Would be useful to have FAQ information regarding income/credit/housing voucher/section 8 information

Special

  • Aside from displaying all available information regarding a property once clicked, you can apply for rental directly through Lovely. Once the form has been filled, Lovely reaches out to the landlord and passes along the application information immediately after submission, giving Users a competitive edge.

Apartments.com

Tech

  • Offers immaculate 3D tours of listings with a combination of Photographs taken at various points within the listing and WebGL to model the area in the browser
  • Uses Bootstrap, jQuery, Google Maps, WebGL
  • Can easily and quickly add a listing with or without a user account
  • User accounts allow users to save favorite listings, search queries, but not share
  • Viewing listing-specific page includes virtually every ounce of information Apartments.com could find on the lsiting itself, including relative locations of schools, restaurants, grocery stores, banks, and other local commodoties

Design

  • Every listing includes floorplans-- crucial information for a user
  • In event of a search yieling no results, instead of suggesting more "similar" options, they simply tell you there's nothing that meets the criteria. It seems obvious, but we currently suggest alternatives, which has led to many occaisions where Users have been left frustrated and confused. This blunt method may save us more headache going forward.
  • Strong and consistent use of omnibox in lieu of dropdowns, something to look at closely since our current User-base is consistently using our support channel to ask questions regarding our filters ( drop-downs )
  • Listings including scheduling availability - small but useful service
  • Average apartment search website with clean UI, though based on poor choices. Obviously not designed for mobile

Special

  • Branded tours of the community / area of each listing, enforcing and ensuring quality as a brand trait

Redfin

Design

  • Right-hand list-view is an elegant display of information in stark contrast to Apartments.com which makes every effort to make everything on screen important. Further information can be accessed once clicked on specific listing
  • Displays school zones within searched address as a standard piece of information for every search ( with detailed school statistics )
  • Heavy use of omni-box with few options that require dropdowns that yield more specific results than Apartments.com
  • Redfin specific agents to do tours for a quick guide to the listing and the community; spark-notes for home searching

Tech

  • "Download All" feature; seemingly broken presently. Pressing button downloads an excel spreadsheet devoid of housing information but with titles like "Price”, “Address”, “Beds”, “Baths”. If the excel output could be done correclty, this could be a huge and easy service to provide, as many people still like to have paper-in-hand when looking at new properties
  • Can contact leasing agent directly through Redfin
  • Toll-free # to call for assistance + additional info ( Twillio )
  • Uses React, Webpack, and Node.js ( catch-22; everything, including large images, loads quickly but reacts very poorly when scrolling quickly )

Special

  • Almost all of the above features

ApartmentShowcase

Design

  • Purple markers for deals; deals page not kept up-to-date

Tech

Special
